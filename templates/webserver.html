HISTORY

Web Browser

Browsers:

The World Wide Web (WWW), commonly known as the Web, is an information system where documents and other web resources are identified by Uniform Resource Locators (URLs, such as https://example.com/), which may be interlinked by hyperlinks, and are accessible over the Internet. The resources of the Web are transferred via the Hypertext Transfer Protocol (HTTP), may be accessed by users by a software application called a web browser , and are published by a software application called a web server.

Tim Berners-Lee is credited with developing, in 1990, both the first web server, and the first web browser, called WorldWideWeb and later renamed Nexus .

Many others were soon developed, with Marc Andreessen's 1993 Mosaic (later Netscape) , being particularly easy to use and install, and often credited with sparking the internet boom of the 1990s.

Today, the major web browsers are Chrome, Safari, Internet Explorer, Firefox, Opera, and Edge.

Web Server and HTTP

A web server is computer software and underlying hardware that accepts requests via HTTP, the network protocol created to distribute web pages, or its secure variant HTTPS. A user agent, commonly a web browser or web crawler, initiates communication by making a request for a specific resource using HTTP, and the server responds with the content of that resource or an error message. The server can also accept and store resources sent from the user agent if configured to do so.

-HTTP is an application layer protocol for transmitting hypermedia docs such as HTML over internet. HTTP is the foundation of data communication for the WWW. Initiated by Tim Berners-Lee in 1989. Different versions HTTP/1(1997), HTTP/2(2015), HTTP/3 came.

HTTP functions as request-response model in client server computing model. Web Browser ...(Client) submits the HTTP request message to the Web Server (hosting the application). Submission is via TCP to a port on a server. Server listening to the port provides resources or performs functions and return a response message. HTTP by default runs on port 80 and if its HTTPS then the port 443.HTTP generic message - start line, header, An empty line, optionally message body. Message REQUEST is Get/hello.htm HTTP/1.1, header, Message Body. Message RESPONSE is HTTPS/1.1 200 OK, header, Message Body. Code servers can use all methods. REST uses the HTTP request types (POST, GET, PUT, and DELETE) to allow users to Create, Read, Update, and Delete (CRUD) via an API.HTTP response status codes indicate whether a specific HTTP request has been successfully completed. Response codes --> 100-199(Info response), 200-299- (Successful responses), 300-399 (redirects ex. after filling the form the page is redirected to thank you page or error page), 400-499 (Client errors- common 401, 403), 500-599 (Server errors). 1990 - Tim Berners Lee wrote two things (1) A Web Browser called WorldWideWeb (2) The world's first Web Server, later known as CERN httpd, which ran on NeXTSTEP. 1995 Apache development began based on NCSA HTTPd server. Apache played a key role in initial growth of WWW became dominant by overtaking HTTPd. 2009 – Apache became the first web server software to serve more than 100 million websites. HTML and CSS

(Hypertext Markup Language defines the structure of a web page) and CSS (cascading Style Sheets -styles the web page or its about presenting the contents).The goal of HTML from its inception in 1990 was to separate document structure from layout.

Tim Berners-Lee wrote first HTML in 1993 but he did not publish the syntax for the style sheets. Lie proposed CSS in 1994 who was working with Tim Berners Lee.1993 - NSCA MOSAIC (browser that made web popular - but it allowed only to change colors).

HTML 1.2 - was simple if web designers want to add content features that HTML did not support, they would have to do with non-standard proprietary technologies such as Adobe Flash. No DTD.

HTML 4.01 - became the ... official standard in 1999. This needs DTD. The original purpose of the doctype was to enable parsing and validation of HTML documents XHTML became the official standard in 2000 (XML version of HTML but with stricter rules for interoperability feature). HTML 5.0 (last final version), does not define DTD. HTML5 enriches the semantic content of documents, new page structure elements such as main, section, article, header, footer, aside, nav, and figure are added. Other major differences between HTML 1.2, HTML 4.01 and HTML5 are Style (presenting the contents) is available in HTML 4.01 and HTML5, JavaScript to make pages respond to user behaviors supported in HTML 4.01 & HTML5 and Audio, Video, Canvas are available in HTML5 only

Sessions

A session ID is a unique number that a Web site's server assigns a specific user for the duration of that user's visit (session). Request has a session-Id to determine the logged in user. Issue if we lose the scalability as all my logged-in user gets data from the WS who store the session-ID. Reading and writing session data at each user interaction must be done without hurting the user experience while the user session is live, no round-trip to the central database should be needed.

Sticky Session

Sticky session refers to the feature of many commercial load balancing solutions for web-farms to route the requests for a particular session to the same physical machine that serviced the first request for that session.

Reverse Proxy

There are multiple reasons for Reverse Poxy (typical rationales are due to high availability, load balancing, security and centralized authentication/authorization).it is critical in theses implementations that layout, design and architecture of the backend infrastructure (those servers which actually handles the requests) are insulated and protected from the outside as far as client is concerned, the reverse proxy is the sole source of all content. Reverse Proxy Server - Idea is they itself does not generate/ host the content but the content is obtained by several backend servers, which normally have no direct connection to the external network.
